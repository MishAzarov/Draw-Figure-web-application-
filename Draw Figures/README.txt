Логика работы:
	Пользователь вводит описание фигур(одна фигура в одной строке), нажимает на
кнопку "Нарисовать". При нажатии на нее, по очереди начинает считываться каждая строка:
проверяется корректность введенной формы, считываются основные параметры фигуры(точки, радиус(ы)) и
дополнительные,если таковые имеются(цвет, заливка). Далее начинается визуализация:
Если введенные формы не прошли валидацию, фигуры не рисуются, а выводится предупреждение
о том, что форма введена не верно; если введенные формы прошли валидацию, то фигуры рисуются в той
последовательности в которой они были введены.
Дополнительные параметры:
	Цвет: -c rgb(..., ..., ...)
		Если таковой не введен, по умолчанию линии фигур будут нарисованы черным цветом
	Заливка: -b rgba(..., ..., ..., ...)
		Если параметр не введен, по умолчанию фигура не заливается
Проверено на данных из ТЗ:
line -p [50, 50] [100, 100] -c rgb(255, 0, 0)
rectangle -p [50, 50] [100, 100] -c rgb(0, 255, 0) -b rgba(255, 0, 0, 0.3)
triangle -p [50, 50] [50, 100] [100, 100] -c rgb(0, 0, 255) -b rgba(255, 0, 0, 0.3)
circle -p [75, 75] -r 25 -c rgb(255, 0, 0) -b rgba(0, 255, 0, 0.3)
ellipse -p [75, 75] -r1 50 -r2 25 -c rgb(0, 255, 0) -b rgba(255, 0, 0, 0.3)
